# Dota 2 High-MMR Pub Analytics

A self-hosted pipeline that automatically spectates high-MMR Dota 2 public matches via the Game State Integration API, collects per-player data, and generates a static analytics dashboard.

The project grew out of wanting to study hero and item trends specifically in top-leaderboard games, which aren't covered well by public trackers. All data collection runs locally on a Windows PC with Dota 2 open.

---

## What it does

1. **Spectates matches automatically** — `spectator.py` clicks through the Watch tab, enters games that haven't been recorded yet, and exits when collection is done.
2. **Receives GSI data** — `server.py` runs a local Flask server that Dota's Game State Integration posts to. Each update is enriched with leaderboard rank data and saved to a CSV.
3. **Analyses the data** — `analytics.py` filters matches by game duration and average leaderboard rank, then outputs hero and player stat tables.
4. **Generates a static site** — `generate_site.py` reads the CSV outputs and injects data into `template.html`, producing a self-contained `index.html` with sortable hero/player/match/item tables.

Optional extras:
- `dota_obs.py` — tracks your own MMR and session stats, writes text files for OBS text sources, and switches scenes during hero selection to hide your pick from stream snipers.
- `watchdog.py` — restarts Dota 2 if it hangs or hasn't been rebooted in the last hour.
- `scrape_ranks.py` / `check_dotabuff.py` — backfill leaderboard ranks and cross-reference match IDs with Dotabuff.

## Setup

### Requirements

- Python 3.11+
- Dota 2 with Game State Integration enabled (see below)

```bash
pip install flask flask-socketio pandas cloudscraper beautifulsoup4 tqdm pyautogui psutil
```

For OBS integration only:
```bash
pip install obs-websocket-py
```

### Enabling GSI in Dota 2

Create a config file at:
```
<Steam>\steamapps\common\dota 2 beta\game\dota\cfg\gamestate_integration\gamestate_integration_analytics.cfg
```

Contents:
```
"GSI Config"
{
    "uri"           "http://localhost:3000/dota"
    "timeout"       "5.0"
    "buffer"        "0.1"
    "throttle"      "0.1"
    "heartbeat"     "30.0"
    "data"
    {
        "map"       "1"
        "player"    "1"
        "hero"      "1"
        "items"     "1"
    }
}
```

### Icon directories

The site uses local icon files that aren't included in the repo due to size. Create these directories next to the scripts and populate them:

| Directory | Contents |
|---|---|
| `hero_icons/` | Hero portrait PNGs, named like `120px-invoker_icon_dota2_gameasset.png` |
| `dota2_icons/` | Item icon `.webp` or `.png` files, named by item key (e.g. `blink_dagger.webp`) |
| `facet_icons/` | Facet icon PNGs, keys come from `facet_icon_map.json` |
| `role_icons/` | Role icons: `1.png` through `5.png` and `sup.png` |

Hero and item icons can be exported from the Dota 2 game files or downloaded from the [Dota 2 Wiki](https://www.dota2.com/wiki).

### Leaderboard data

`server.py` reads `processed_leaderboard.csv` (columns: `Никнейм`, `Место в ладдере`, `Команда`) to annotate players with their leaderboard rank at collection time. The included `acc_id_ld.csv` and `nickname_ld.csv` serve as the rank lookup tables for `rank_merger.py`.

## Data collection workflow

```bash
# Terminal 1 — start the GSI receiver
python server.py

# Terminal 2 — start the auto-spectator
# (Dota 2 must be open on the Watch tab)
python spectator.py

# Optional: keep the game from crashing overnight
python watchdog.py
```

Match data accumulates in `dota_matches_history.csv`. When you want to generate the site:

```bash
# Fill in missing leaderboard ranks from the lookup CSVs
python rank_merger.py

# Compute hero and player stats (filters by duration and rank)
python analytics.py

# Build the static site
python generate_site.py
# → writes index.html, open it in any browser
```

## Site features

The generated `index.html` is fully self-contained — no server needed, just open it in a browser.

- **Heroes tab** — sortable by games, win rate, KDA, average net worth, and a Wilson Score rating (confidence-adjusted win rate that demotes heroes with few games). Positions 4 and 5 are merged into a single "Support" column. Clicking a row opens a modal with per-facet and per-position breakdowns.
- **Players tab** — filterable by name, hero, or account ID. Clicking a row shows full match history for that player within the dataset.
- **Matches tab** — expandable cards showing full rosters, items, and KDA. Searchable by hero or player name.
- **Items tab** — item win rates filterable by role, net-worth bracket, and whether to include neutral items.
- **Global filters** — rank bracket and minimum game duration sliders that recompute all stats client-side without a page reload.

## Coordinate calibration for `spectator.py`

The auto-spectator uses pixel coordinates that depend on your monitor resolution and Dota 2 window position. Edit the constants at the top of the file:

```python
WATCH_TAB          = (850, 200)      # Watch tab button
MATCH_LIST_X       = 530             # X coordinate of the match list
MATCH_LIST_START_Y = 345             # Y of the first match row
MATCH_ROW_STEP     = 60              # Pixels between rows
CHECK_PIXEL        = (1040, 582)     # Pixel inside the "WATCH IN-GAME" button
PANEL_COLOR        = (222, 229, 244) # Button colour when the panel is visible
```

To find the right values for `CHECK_PIXEL` and `PANEL_COLOR`, use `pyautogui.pixel(x, y)` to sample the colour at the button location on your screen.

PyAutoGUI's failsafe is enabled: moving the mouse to any screen corner stops the script immediately.

## Notes

- All data stays local. The only outbound requests are the scraping scripts hitting Dotabuff.
- Match records where net-worth leader and Radiant Win Chance disagree are excluded from the site — they were captured during a comeback, before the lead changed.
- The Wilson Score for hero ratings is the lower bound of a 95% confidence interval on win rate. It naturally ranks heroes with few games below those with more data, even if their raw win rate looks high.
- `spectator.py` requires Dota 2 to be running in windowed or borderless-windowed mode with the Watch tab visible and no other windows on top.
